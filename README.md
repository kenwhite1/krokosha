# Крокоша 🐊

Уютная игра в ассоциации (цифровой «Крокодил») для Telegram Mini Apps. Один
игрок становится ведущим: ему выпадает тайное слово, и он по одной открывает
подсказки. Остальные наперегонки пишут догадки. Кто угадал первым, забирает
очки, и чем меньше подсказок ушло, тем больше награда.

Полностью на русском, в фирменном тёплом стиле (как «Однушечка» и tt).

## Возможности

- **Одиночная игра** против ботов, три уровня сложности (движок крутится прямо в браузере).
- **Быстрая игра** — публичный подбор; пустые места занимают боты, неотличимые от живых игроков.
- **Игра с друзьями** — общая комната по коду из четырёх символов; пустые места занимают видимые боты.
- Боты умеют и вести (открывать подсказки), и угадывать; тайминги человечные и побиваемые.
- Рейтинг, профиль, монеты, серии побед.
- Telegram-бот с командами `/start /play /rules /about /help`.

## Архитектура

Монорепо (npm workspaces):

- `shared/` — чистый детерминированный движок игры (`engine.ts`, `view.ts`,
  `bots.ts`, `words.ts`, `rng.ts`, `types.ts`). Используется и клиентом, и сервером.
- `server/` — Hono + better-sqlite3 + grammy (Telegram-бот) + jose (JWT).
  Авторитетные онлайн-комнаты в памяти (`rooms.ts`), REST API (`api.ts`),
  раздаёт собранный клиент.
- `app/` — React + Vite + Zustand. Соло-драйвер и онлайн-драйвер (опрос комнаты)
  живут в `store.ts`; экраны в `app/src/screens/`.

Соло-игра работает офлайн (движок на клиенте); онлайн — на сервере, клиент
опрашивает `GET /api/room/:code` примерно раз в 1.1 секунды.

## Разработка

```bash
npm install
npm run dev:server   # сервер на :3000 (DEV MODE без BOT_TOKEN: авторизация-заглушка)
npm run dev:app      # vite на :5175, проксирует /api на :3000
```

Без `BOT_TOKEN` сервер поднимается в DEV MODE: авторизация подставляет одного
локального пользователя — удобно для локальной игры. Соло-режиму сервер не нужен.

Проверки:

```bash
npm test         # юнит-тесты движка
npm run typecheck
npm run build    # сборка клиента в app/dist
```
